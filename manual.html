Introduction

Journal Club Manager is a web-application designed to help labs in managing their journal clubs and presentations. Members can easily create an account and from there submit, edit and delete their presentations, or suggest papers for future sessions.

Journal Club Manager comes with a complete administration interface allowing admins and organisers to manage the sessions (their date, type, time, number of presentations, etc.), to post news, send emails to the other members, create group of members and a lot more!

Finally, Journal Club Manager relies on several automated features that make the organization of journal clubs an easy and effortless task.

  • Mailing system: Journal Club Manager includes a mailing system to notify members about upcoming journal club sessions, recent news or the last suggestions added by the other users.
  • Automatic assignment: Journal Club Manager can automatically assign members as speaker for the future sessions
  • Automatic backup: Journal Club Manager can automatically update databases and application files

System Requirements

  • A web server running PHP 5.4 or later (with mcrypt extension installed and activated)
  • MySQL (5.0 or later)
  • CRON table (on Linux servers) or scheduled tasks (on Microsoft Windows servers) *required for email notifications
  • SMTP server (or a Google Mail account)

Installation

0. Upgrading from version 1.2.1 or earlier

The Journal Club Manager has sensibly changed since its very first versions. If you are running with one of these versions, you can upgrade to the newest version of the JCM but you will need to follow a specific procedure:

  1. Save your previous config file (path_to_JCM/admin/config/config.php) somewhere outside the application.
  2. Overwrite all the JCM's folders already present on your server EXCEPT the "uploads" folder.
  3. Copy your previous config file (the one you saved in step 1) to the new config folder (path_to_JCM/config/config.php).
  4. Update your JCM version by running http://www.mydomain.com/jcm/install.php and follow the steps described in the next sections.
  5. Congrats!

1. Upload files on the server

Using GIT

Simply clone the Journal Club Manager repository ( https://github.com/Fperdreau/jcm.git) at the root of your web-server.

Using FTP

  1. Download JCM here
  2. Unzip the archive
  3. Upload the “jcm” folder to the root of your web-server using your favorite FTP client (e.g. FileZilla).
  4. Make sure that you have the writing permissions for all the folders and files (set folders chmod to 0755 and files chmod to 0644).

2. Preparation of the database

Go to your SQL database interface (e.g. PHPmyadmin) and create a new database (if you have the rights, however, ask your admin for the name of the database). That’s it!

3. Online installer

In your favorite web-browser, go to the URL: http://www.mydomain.com/jcm/install.php.

Step 0: New installation or update

The installer will automatically check for the presence of previous installations of the Journal Club Manager.

If no previous installation is found, then you will only have the option to start a new installation. This will create all the required SQL tables, folders and files.

Otherwise, you will also have the option of updating a previous installation. Do not worry, this will not overwrite your preexistent data!

Step 1 : Database configuration

Hostname: Your SQL hostname (e.g.: sql.mydomain.com)

User name: your SQL username (the one you may use to access PHPMyAdmin).

Password: your SQL password (The same you are using to access PHPMyAdmin).

Database name: the name of the database you have created at the Step 3 of the preparation phase.

Database prefix: Choose a prefix that will be used to create tables in your database (e.g.: jcm).

*Make sure the prefix you choose if not already used by other applications installed on your server.

Step 2: Application configuration

Many features of Journal Club Manager requires the ability to send emails (notifications, reminders or assignments to users)

To use these features, you need to set the mailing system and particularly the SMTP host. You can either choose to use your own SMTP server (if there is) or to use the one of any email provider (GMAIL, OUTLOOK, etc.).

Send Email address: your journal club email

Sender Name: Name associated to the email address

Email host: SMTP host address (e.g. smtp.gmail.com if you are using a GMAIL address).

SMTP access: Security protocol (SSL/TLS/none).

*Note that using SSL/TLS requires that SSL PHP extension is installed on your web-server.

Email username: your email username

Email password: your email password.

Step 3 : Creation of the admin account (only for new installation)

Username: choose a username for your admin account

Password: choose a password for your admin account

Confirm password: re-enter your password

Email: provide an email address that will be associated with the admin account.

Step 4: Delete installation files

For security reasons, we recommend to delete installation files from your web-server. Go to your FTP and delete the “install.php” file located at the root of the “jcm” folder.

Step 5 (optional): Set plugins

Go to Admin>Plugins. Here you can install JCM plugins and modify their settings.

Step 6 (optional): Set automatic scheduled tasks (e.g. email notification)

Go to Admin>Scheduled tasks. Here you can install and set scheduled tasks.

My Tools

Submission

JCM members have the possibility to either submit a new presentation (Submit), to suggest a presentation to other members (Make a suggestion) or to submit a presentation suggested by other members (Select a suggestion).

Submit or suggest a presentation

  • Type: Select a presentation type. Default JCM presentation types are: paper (research article), research (user's own research), methodology (methodology discussion), guest (guest speaker).
  • Speaker (for guest's presentation onky): Name of the invited speaker
  • Date: Date of your presentation
  • Title: Your presentation title
  • Abstract: Your presentation abstract
  • Authors (for paper presentation only): Authors of the paper
  • File: Files associated to the presentation (pdf, slides, etc.)

Add a suggestion

The procedure for making a suggestion (suggest a presentation) is similar to submitting a new presentation except that the user does not have to provide a date (see Submit a new presentation).

Select a paper from the suggestions list

You can choose to submit a presentation suggested by other members. To do so, click on "Select a suggestion" in submission menu. You will be prompted with a selection list of all suggested presentations. Select the one you wish to present, and choose a date for your presentation. You can also edit the presentation information and/or add/delete files attached to this presentation.

Send emails

You can send an email to other JCM members from My Tools>Email .

  • Attach files (optional): You can either drag & drop files you want to attach to your emails or upload them by clicking on the "Add Files" buttons
  • Select recipients: In the selection list, select the members (one at a time) to whom you want to send your email. If you want to add all of the JCM members to the recipient list, then simply select "All" from the selection list. Selected members appear in the recipients field. You can remove previously added members by clicking on the trash icon next to the member's name
  • Hide recipients: if set to "yes", then recipients email addresses and identities will not be visible. Note that this will also prevent recipients to "reply to all".
  • Add as news: if set to "yes", then email will be also published as news. This option is automatically set to yes when the recipients field is set to "all". Note that in this case, it is still possible to manually set this option to "no".
  • Subject: Your email's subject
  • Message: Your email's content

Add a news

Journal Club Manager includes a blog-like section ("News", accessible from the main menu) where members can post announcements or news that might be interesting to the other members or visitors. The latest news are also displayed on the application home page and can be included in the weekly digest email. News can be posted and edited from My Tools>News.

You can either add a new post or edit an already existing one. Previous posts are accessible from the selection list.

  • Title: Your post's title (255 characters length maximum)
  • Homepage: Display your post on the home page
  • Message: Content of your post.

Once every field is filled in, you can click on the "Add" button to post your news.

Suggestions

Here, you can find all suggestions and download the associated files. Archives can be filtered by year.

My profile

Every user can access his/her own profile page and modify his/her account information, manage submissions or availabilities.

My information

Here, you can find and modify your personal and login information.

Contact information

Here, you can edit your contact information and the way you want to interact with Journal Club Manager.

  • Email: Your email address
  • Email notifications: If set to "Yes", then you will receive notifications from Journal Club Manager by email (e.g. new submissions, etc.)
  • Reminders: if set to "Yes", then you will receive reminders about upcoming journal club sessions and about your own presentations
  • Assignment: if set to "Yes", then you allow Journal Club Manager to automatically assign you to future sessions as speaker

My Availability

Here you can inform Journal Club Manager about the dates on which you are not available for presenting.

To do so, simply click on a date of your choice.

How do I change my password?

From Profile>Personal information, click on the link "change my password".

An email will be automatically sent to your email address with further instructions.

How do I delete my account?

To delete your account, simply click the big red "Delete my account" at the bottom of your profile page.

You will be prompted with a dialog to provide your login information.

Finally, click on "Delete my account" to confirm the operation.

Deleted accounts cannot be recovered. All your account information will be lost.

Organizer

Session manager

Here you can find information relative to the journal club (address, time, map) and contact the organizers by email.

  • Session settings: Here you can specify the meeting room, journal club time and day, as well as the number of presentations per session.
  • Session/Presentation: You can set the default meeting type (e.g. journal club, business meeting), and add or delete session and presentation types
  • Manage sessions: Modify information of individual sessions (session type, time, or speakers). If a new speaker is selected, this will automatically cancel the assignment of the previously selected speaker. She or he will then receive a notification email about this change.

Reminder & Digest emails manager

Reminder & Digest emails manager are accessible from Organizer> Reminder and Organizer>Digest respectively.

The reminder and digest emails manager allows you editing the content of reminder and digest emails. From these pages, you can select which and how information should appear in the emails.

Once you have validated your modifications, you can preview the email content by clicking on the Preview button.

Assignments

Automatic assignments of speakers can be monitored from Organizer>Assignments

Plugins

List of available operations:

Uninstall plugin
Show plugin's settings

List of built-in plugins:

AutoAssignment: Automatically assigns members of the JCM (who agreed upon being assigned by settings the corresponding option on their profile page) as speakers to the future sessions. The number of sessions to plan in advance can be set in the plugin's settings.

Groups: Automatically creates groups of users based on the number of presentations scheduled for the upcoming session. Users will be notified by email about their group's information. If the different groups are meeting in different rooms, then the rooms can be specified in the plugin's settings (rooms must be comma-separated).

Scheduled tasks

Journal Club Manager relies on scheduled tasks in order to automatically send notifications, reminders or assignments by email, or simply to backup files and database.

In order to do so, you need to set CRON jobs on your server (if your server is running on Linux) or scheduled tasks (if it’s running on MS Windows) pointing to the PHP script 'run.php' located in the 'jcm/cronjobs' folder and to make it run every hour.

Here are tutorials to automatically run PHP file:

  • on MS Windows: https://www.drupal.org/node/31506

  • on Linux: http://www.thegeekstuff.com/2011/07/php-cron-job/

Once done, scheduled tasks can be managed from the Journal Club Manager (Admin>Scheduled Tasks).

How to schedule a task

First, the task must be installed and activated. Then, in Date & Time section, select a date and a time when the task should be executed.

You also have the possibility to repeat the task automatically. To do so, in Frequency, define how often the task should be executed (every x month, x days, x hours, and x minutes).

Finally, click on the "Update" button to save your settings

List of available operations:
Install task
Uninstall task
Show task's settings
Activate/deactivate task
Execute scheduled task

List of built-in scheduled tasks:

Mailing: send weekly digest including the last news, details about the upcoming session, the list of the future sessions and the list of the newest suggestions.

Notification: Send list of last submissions by email.

Reminder: send reminder for the upcoming session.

SpeakerAssignment: automatically assigned speakers to each presentation by pseudo-randomly picking up one organizer. The simple rule is that all organizers must have been a chair once for this particular session type (e.g. journal club, group meeting, etc.) before an organizer can be selected again as a chair.

MakeGroup: Split users into groups for one session (1 group per presentation)

DbBackup.php: do a backup copy of the database and save it into a .sql file that can be found in “jcm/backup/mysql” folder.

FullBackup: do a backup copy of the database and of all the files and store it in an archive that can be found in “jcm/backup/complete” folder.

MailSender: Checks whether all emails have been sent and sends them otherwise. It also cleans the mailing database by deleting the oldest emails. The number of days of email storage can be defined in the task's settings (default is 10 days).

Create an account

  1. First, go to your Journal Club Manager page.
  2. Click on the sign up icon (top-right of the page)
  3. You will be prompted with a form. Simply fill it in with your information (all fields are required)
  4. Click on the sign-up button to submit your form. You're done!

You will receive a confirmation email as soon as your account has been activated by the admins or organizers.

Once done, you can login to your Journal Club Manager by clicking on the login button (top-right of the page) and provide your login information.
